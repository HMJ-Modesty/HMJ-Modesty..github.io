# 1.5.1 面向对象三大特性

  • 封装

    • 将属性和方法写到类的里面的操作即为封装
    • 封装可以为属性和方法添加私有权限

  • 继承

    • 子类默认继承父类的所有属性和方法
    • 子类可以重写父类属性和方法

  • 多态

    • 传入不同的对象，产生不同的结果

# 1.5.2 多态

# 1.5.2.1 了解多态

多态指的是一类事务有多种形态，（一个抽象类有多个子类，因而多态的概念依赖于继承）

  • 定义：多态是一种使用对象的方式，子类重写父类方法，调用不同子类对象的相同父类方法，可以产生不同的执行结果
  • 好处：调用灵活，有了多态，更容易编写出通用的代码，做出通用的编程，以适应需求的不断变化！
  • 实现步骤：
    • 定义父类，并提供公共方法
    • 定义子类，并重写父类方法
    • 传递子类对象给调用者，可以看到不同子类执行效果不同

# 1.5.2.2 体验多态

# 1.5.3 类属性

# 1.5.3.1 设置和访问类属性

  • 类属性就是 类对象所拥有的属性，它被 该类的所有实列对象所共有
  • 类属性可以使用 类对象或 实列对象访问

类属性的优点

  • 记录的某项数据始终保持一致时，则定义类属性
  • 实列属性 要求每个对象为其单独开辟一份内存空间 来记录数据，而类属性为全类所共有，仅占用一份内存，更加节省内存空间

# 1.5.3.2 修改类属性

类属性只能通过类对象修改，不能够通过实列对象修改，如果通过实列对象修改类属性，表示的是创建了一个实列属性

# 1.5.4 类方法和静态方法

# 1.5.4.1 类方法

  • 需要用装饰器 @classmethod 来标识其为类方法，对于类方法，第一个参数必须是类对象，一般以 cls 作为第一个参数
# 1.5.4.1.1 类方法使用场景
  • 当方法中 需要使用类对象 (如访问私有类属性等) 时，定义类方法
  • 类方法一般和类属性配合使用

# 1.5.4.2 静态方法

# 1.5.4.2.1 静态方法特点
  • 需要通过装饰器 @staticmethod 来进行修饰，静态方法既不需要传递类对象也不需要传递实列对象 (形参没有 self/cls)
  • 静态方法也能够通过 ** 实列对象 ** 和 类对象 去访问
# 1.5.4.2.2 静态方法使用场景
  • 当方法中 既不需要使用实列对象 (如 实列对象，实列属性)， 也不需要使用类对象 (如类属性，类方法，创建实列等) 时，定义静态方法
  • 取消不需要的参数传递 ，有利于 减少不必要的内存占用和性能消耗
